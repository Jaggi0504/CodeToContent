🚀 **Automated README & LinkedIn Post Generator for Jupyter Notebooks** 🚀  

I’ve built an end‑to‑end pipeline that turns any `.ipynb` into a polished GitHub README **and** a ready‑to‑publish LinkedIn post with a single function call.

### What it does  
1️⃣ **Extracts** markdown & code cells from a notebook (`load_notebook`).  
2️⃣ **Prompts** Groq’s GPT‑OSS‑120B via LangChain to craft a structured README.  
3️⃣ **Feeds** that README into a second prompt to generate a concise, technical LinkedIn summary.  
4️⃣ **Orchestrates** the two steps with LangGraph’s state‑graph, then writes `README_generated.md` & `linkedin_post.txt`.

### Core stack  
- **Python 3.9+** – core logic  
- **nbformat** – notebook parsing  
- **LangChain + LangChain‑Groq** – prompt templating & LLM abstraction  
- **LangGraph** – workflow graph (generate_readme → generate_linkedin_post)  
- **Groq LLM (gpt‑oss‑120b)** – high‑throughput generation  
- **Mermaid** (via `draw_mermaid_png`) – visual graph rendering  

### Why it matters  
Documentation and social‑media outreach are often after‑thoughts that drain developer time and become stale. By automating both artifacts, we get:  
- Consistent, up‑to‑date READMEs that cover title, problem, approach, tech stack, results, and next steps.  
- Technical LinkedIn posts that highlight the stack, the problem solved, and invite community discussion—all without manual copy‑pasting.
